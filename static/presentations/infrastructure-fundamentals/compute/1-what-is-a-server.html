Infrastructure Fundamentals

The Server Concept

A server provides compute resources that process requests and deliver services over a network

Hardware definition: A physical computer optimized for continuous operation and network accessibility

Software definition: Any program that listens for requests and sends responses (web server, database server, application server)

Both definitions coexist: "database server" refers to both the PostgreSQL process and the machine running it

The Client-Server Model

One system (the server) provides services that other systems (clients) consume

Centralized resource management: A single server maintains the authoritative copy of data and services

Scalability: Adding clients doesn't require duplicating the core serviceâ€”only the server's capacity to handle concurrent requests

Examples: Web browser requests pages from a web server, mobile app queries an API server, backend retrieves data from a database server

Server Deployment Models

The same application logic can run on different infrastructure foundations

Physical Servers: Dedicated hardware, maximum performance, complete control

Virtual Machines: Software-based emulation, better utilization, rapid provisioning

Containers: Lightweight isolation, millisecond startup, efficient scaling

Serverless Functions: Event-driven execution, automatic scaling, no infrastructure management

Physical Servers & Virtual Machines

Physical Servers (Bare-Metal): OS runs directly on hardware, maximum performance, operational overhead (procurement, installation, maintenance)

Virtual Machines: Hypervisor layer manages hardware, multiple VMs on same host, each VM has its own OS instance

VM Benefits: Better hardware utilization, rapid provisioning (minutes vs weeks), consumption-based pricing in cloud platforms

VM Trade-offs: 2-10% performance overhead, full OS weight per instance, OS-level maintenance for each VM

Containers & Serverless

Containers: Package application with dependencies, share host OS kernel, isolated processes

Container Benefits: Lightweight (millisecond startup), consistent environments across dev/test/prod, efficient scaling with orchestration (Kubernetes)

Serverless Functions: Code executes in response to events (HTTP request, queue message, timer), platform manages all infrastructure

Serverless Benefits: No servers to manage, automatic scaling, pay-per-execution billing, ideal for variable workloads

Serverless Constraints: Execution time limits (5-15 min), cold start latency, best for stateless operations

Server Roles

Web Servers: Accept HTTP/HTTPS requests, serve static files or proxy to application servers (nginx, Apache)

Application Servers: Execute business logic, generate dynamic content, process user requests (Gunicorn, uWSGI for Python)

Database Servers: Manage persistent data storage, enforce integrity, optimize queries, handle concurrent access (PostgreSQL, MySQL)

File Servers: Provide centralized storage over network, implement access controls (NFS, SMB, or cloud object storage)
