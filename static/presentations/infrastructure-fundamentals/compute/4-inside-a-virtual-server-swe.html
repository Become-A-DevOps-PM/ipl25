Förstå virtualiseringsarkitektur

Vad är virtualisering?

En virtual server (VM) är en mjukvarubaserad emulering av en fysisk dator som kör sitt eget operativsystem och applikationer

Från mjukvarans perspektiv framstår VM:en som en dedikerad fysisk maskin med fullständig isolering från andra VMs

Flera VMs kan dela samma fysiska hårdvara, vilket förbättrar resursanvändningen och minskar kostnaderna

Hypervisor-lagret

Hypervisor hanterar fysiska hårdvaruresurser och allokerar dem till virtuella maskiner

Type 1 (bare-metal): Körs direkt på hårdvaran med bättre prestanda - används i produktion (VMware ESXi, Hyper-V, KVM)

Type 2 (hosted): Körs ovanpå ett OS med enklare uppsättning - används för utveckling (VirtualBox, VMware Workstation)

Cloud-plattformar använder Type 1 hypervisors: Azure använder Hyper-V, AWS använder modifierad KVM

Virtuella komponenter

vCPUs: Virtuella processorer som tidsdelas på fysiska CPU-kärnor - fler vCPUs möjliggör fler samtidiga operationer

Virtual Memory: Allokerad del av fysiskt RAM som är isolerad per VM - att få slut orsakar disk-swapping och allvarlig prestandaförsämring

Virtual Storage: Diskar som egentligen är filer på hosten (VMDK, VHD/VHDX, qcow2) - prestanda beror på underliggande lagringstyp

Virtual NICs: Nätverksgränssnitt som kopplar VMs till virtuella nätverk med tvingad isolering och säkerhetsregler

Fysiska vs virtuella servrar

Resursallokering: Fysiska servrar dedikerar alla resurser till en arbetsbelastning - Virtuella servrar delar och multiplexar fysiska resurser

Provisioneringshastighet: Fysiska servrar tar dagar eller veckor att driftsätta - Virtuella servrar provisioneras på minuter från resurspooler

Skalning: Fysisk kräver hårdvaruanskaffning - Virtuell möjliggör elastisk skalning genom konfigurationsändringar

Kostnadsmodell: Fysisk använder kapitalutgifter (CapEx) - Virtuell använder driftskostnader (OpEx) fakturerade efter användning

Virtual Servers i Azure

Azure hanterar den fysiska infrastrukturen - hypervisors, servrar, nätverk och lagringssystem

Du specificerar: Region, Size, OS Image, Storage-diskar och Virtual Network-konfiguration

VM-serier: B-series (burstable), D-series (general-purpose), E-series (memory-optimized), F-series (compute-optimized)

Managed Disks: Standard HDD (säkerhetskopior), Standard SSD (allmänna arbetsbelastningar), Premium SSD (produktionsdatabaser)

Avvägningar med virtualisering

Performance overhead: Hypervisor förbrukar resurser och adderar 2-10% latens - I/O-arbetsbelastningar ser mer overhead än compute-arbetsbelastningar

Noisy Neighbors: VMs på samma host tävlar om resurser - en VMs överdriven användning kan försämra andras prestanda

Abstraktionsgränser: Vissa arbetsbelastningar behöver direkt hårdvaruåtkomst - specialiserade nätverkskort, GPUs eller specifika CPU-funktioner

För de flesta arbetsbelastningar är dessa avvägningar acceptabla - när inte, använd dedicated hosts eller bare-metal-instanser

Virtual servers emulerar fysiska datorer genom mjukvara, vilket möjliggör effektiv resursanvändning och snabb provisionering
